# 数据库系统

## 什么是数据库系统

数据库系统（Database System）研究的是**如何高效、可靠、安全地存储和检索大量数据**。从你的手机通讯录到银行交易记录，从电商商品库存到社交网络的关系图谱——几乎所有软件背后都有数据库。

现代数据库系统不只是"存数据"，它要解决的核心问题包括：如何快速查找（索引）、如何保证并发操作不冲突（事务和锁）、如何在系统故障时不丢数据（持久化和恢复）、如何在大规模数据下仍保持高性能（查询优化）。

## 为什么要学数据库系统

**后端开发的基石。** 不管是 Web 开发、微服务还是 AI 应用，数据存取都是绕不开的环节。不理解数据库，你只能写"玩具级"的应用。

**理解数据存储的本质。** 向量数据库（你 RAG 项目中用的）本质上也是一种数据库，只是索引方式从 B+ 树变成了向量近似搜索。理解了传统数据库，学向量数据库会事半功倍。

**面试必考。** 事务 ACID、索引原理、SQL 优化、数据库设计——这些是后端面试的铁题。

**考研 408 核心科目。** 分值约 25 分，重点考察关系模型、SQL、事务处理、索引结构。

**AI 系统的数据层。** 特征存储、模型元数据管理、训练数据流水线——AI 工程的基础设施也大量使用数据库。

## 核心知识模块

### 关系模型与 SQL

- **关系模型**：表、行、列，关系代数（选择、投影、连接、并/差/笛卡尔积）
- **SQL**：DDL（建表）、DML（增删改查）、DCL（权限控制）
- **高级 SQL**：子查询、聚合函数（GROUP BY/HAVING）、窗口函数
- **数据库设计**：ER 图、范式（第一范式到 BCNF）、反规范化

### 存储与索引

- **磁盘存储**：页（Page）是磁盘 I/O 的基本单位
- **B+ 树索引**：数据库最常用的索引结构，多路平衡树，适合范围查询
- **哈希索引**：O(1) 精确查找，不支持范围查询
- **聚簇索引 vs 非聚簇索引**：数据是否按索引顺序物理存储
- **复合索引与最左前缀匹配**
- **索引的选择性**：高选择性列才适合建索引

### 查询处理与优化

- **查询执行计划**：`EXPLAIN` 命令，理解数据库怎么执行你的 SQL
- **连接算法**：嵌套循环连接、排序-合并连接、哈希连接
- **查询优化器**：基于代价的优化（Cost-Based Optimization）
- **SQL 调优**：避免全表扫描、合理使用索引、避免 SELECT *

### 事务处理

- **ACID 属性**：原子性（Atomicity）、一致性（Consistency）、隔离性（Isolation）、持久性（Durability）
- **并发控制**：锁（共享锁/排他锁）、两阶段锁（2PL）、死锁检测与预防
- **隔离级别**：读未提交、读提交、可重复读、串行化
- **MVCC**：多版本并发控制，InnoDB 的实现方式

### 故障恢复

- **日志（Log）**：Write-Ahead Logging（WAL）— 先写日志再写数据
- **ARIES 算法**：现代数据库的标准恢复算法
- **检查点（Checkpoint）**：加速恢复过程
- **备份策略**：全量备份、增量备份、差异备份

### NoSQL 与新型数据库

- **键值存储**：Redis、DynamoDB — 简单高效，适合缓存和会话管理
- **文档数据库**：MongoDB — 灵活的 Schema，适合半结构化数据
- **列族存储**：Cassandra、HBase — 适合大规模分布式写入
- **图数据库**：Neo4j — 适合关系查询（社交网络、知识图谱）
- **NewSQL**：TiDB、CockroachDB — 兼具 SQL 的一致性和 NoSQL 的扩展性

## 怎么学

**理论和实践并重。** 数据库课程的特点是概念多、实践性强。光看书不写 SQL，概念永远是纸上谈兵。

**从一个具体数据库入手。** 推荐先学 MySQL——文档丰富、社区庞大、广泛使用。装一个 MySQL，自己建表、写查询、看执行计划。

**做 CMU 15-445 实验。** 这门课要求你实现一个完整的数据库系统（Buffer Pool Manager、B+ 树索引、查询执行器、并发控制）。做完这些实验，你对数据库的理解会脱胎换骨。

**学会看 EXPLAIN。** 这是数据库开发最重要的调试工具。每次写完一个 SQL 查询，都用 `EXPLAIN` 看看执行计划是否符合预期。
