# Data Structures and Algorithms

## What are Data Structures and Algorithms

**Data structures** are ways to organize and store data. Arrays, linked lists, stacks, queues, trees, graphs, hash tables — each data structure has its own characteristics and is suited for different scenarios.

**Algorithms** are steps to solve problems. Sorting, searching, traversal, dynamic programming, greedy — each algorithm is a general solution for a class of problems.

Data structures and algorithms are inseparable. The data structure you choose determines which algorithms you can use; the efficiency of an algorithm often depends on the properties of the underlying data structure. For example, binary search is fast on arrays but not on linked lists — because arrays support random access while linked lists do not.

## Why Learn Data Structures and Algorithms

**Fundamental programming skill.** Data structures and algorithms are the foundation of all programming tasks. Whether you do frontend, backend, data science, or AI, you can't avoid them. Without knowing data structures, you might use a list for a task that should use a hash table, resulting in orders of magnitude worse performance.

**Core interview topic.** Algorithm problems are mandatory in technical interviews at major companies. Solving hundreds of problems on LeetCode is almost a standard requirement for job seekers.

**Develop computational thinking.** Learning algorithms isn't just about specific problem-solving techniques — it's about developing the ability to decompose problems, build abstractions, and analyze complexity. This way of thinking is valuable in any technical field.

**Understand AI and systems.** Search engines use graph algorithms, databases use B+ trees, compilers use syntax trees, and recommendation systems use graphs and matrix operations. Without understanding these foundations, you're limited to calling APIs.

## Core Knowledge Areas

### Basic Data Structures

- **Arrays and linked lists**: contiguous storage vs pointer linking, random access vs sequential access
- **Stacks and queues**: LIFO vs FIFO
- **Hash tables**: key-value mapping, collision handling, O(1) average lookup
- **Trees**: binary trees, binary search trees, AVL trees, red-black trees
- **Graphs**: adjacency matrix vs adjacency list, directed vs undirected graphs

### Basic Algorithms

- **Sorting**: bubble, selection, insertion, merge, quicksort, heapsort
- **Searching**: linear search, binary search, BFS, DFS
- **Recursion**: recursive thinking, converting recursion to iteration

### Advanced Topics

- **Dynamic programming**: optimal substructure, overlapping subproblems, state transition equations
- **Greedy algorithms**: conditions for local optimum to global optimum
- **Divide and conquer**: splitting big problems into smaller ones, solving separately, then merging
- **Graph algorithms**: shortest paths (Dijkstra, Floyd), minimum spanning trees (Prim, Kruskal), topological sort
- **Advanced data structures**: heaps, union-find, segment trees, trie trees

### Complexity Analysis

- **Time complexity**: O(1), O(log n), O(n), O(n log n), O(n^2) — using Big O notation to measure algorithm efficiency
- **Space complexity**: how much extra memory an algorithm needs
- **Best/worst/average case**: performance differences under different inputs

## How to Learn

**Learn basic data structures first, then algorithms.** Data structures are the载体 of algorithms. Without understanding linked lists, it's hard to understand linked list sorting; without understanding trees, it's hard to understand tree traversal algorithms.

**Problem-solving is necessary, but think while you solve.** Don't blindly chase quantity. After each problem, ask yourself: what data structure did this problem use? What algorithm? What's the time complexity? Is there a more optimal solution?

**Solve by topic, not randomly.** Finish array and string problems first, then linked lists, then trees, then dynamic programming. Problems within the same topic share patterns, making concentrated practice more effective.

**Write code by hand, don't just read.** Understanding a solution and writing it yourself are different things. Writing code on paper or a whiteboard exposes how well you grasp the details.

**Draw diagrams to aid understanding.** Linked list pointer changes, tree rotations, graph traversals — these processes are much more efficiently understood through diagrams than text alone.

**Textbooks**

- *Algorithms, 4th Edition* (Sedgewick) — Classic textbook with Java implementations, clearly written
- *Introduction to Algorithms* (CLRS) — The most authoritative algorithm textbook, comprehensive but theoretical
- *Grokking Algorithms* — Accessible illustrated introduction to algorithms

- [UC Berkeley CS61B](https://sp21.datastructur.es/) — Berkeley's data structures course with complete Java labs

- [Princeton Algorithms (Coursera)](https://www.coursera.org/learn/algorithms-part1) — Taught by Sedgewick, with Java assignments

**Problem-Solving Platforms**

- [LeetCode](https://leetcode.com/) — The most popular algorithm problem-solving platform
