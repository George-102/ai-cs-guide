# VisuAlgo — Algorithm Visualization Tool

## Introduction

VisuAlgo is a data structure and visualization tool developed by Professor Steven Halim from the National University of Singapore. It's not a traditional "course" but an interactive website that lets you see the **dynamic execution process** of various data structures and algorithms.

On VisuAlgo, you can watch: step-by-step element swaps in sorting algorithms, BST insertion and deletion processes, how Dijkstra's algorithm finds shortest paths one step at a time, how BFS/DFS traverses a graph, how hash tables handle collisions — these animated demonstrations make abstract algorithms tangible and intuitive.

For self-learners, VisuAlgo is one of the best supplementary tools available. After learning an algorithm, watching the animation here helps you connect the code with the algorithm's execution process.

## Prerequisites

None. VisuAlgo is suitable for learners at any stage.

## Difficulty

🌟 (Beginner)

This is a tool, not a course. The barrier to entry is extremely low — just open the website and use it.

## Estimated Study Hours

Use as needed. Recommended: spend 10-15 minutes on VisuAlgo after learning each algorithm to watch the corresponding animation.

## Resources

- Website: [visualgo.net](https://visualgo.net/)
- Languages: Chinese, English, and more
- Topics covered: sorting, linked lists, stacks, queues, hash tables, BST, AVL trees, heaps, graphs, union-find, dynamic programming, and more

## Self-Study Tips

1. **Watch the animation right after learning an algorithm.** For example, after learning quicksort, go to VisuAlgo and watch a complete sorting animation — you'll gain a whole new understanding of the partition process.

2. **Input your own data.** VisuAlgo lets you enter custom data and watch how the algorithm processes it. This is more educational than watching preset demonstrations.

3. **Use alongside problem-solving.** When you encounter an algorithm problem on LeetCode that you can't solve, go to VisuAlgo first to watch the corresponding algorithm demo, then come back to the problem.

4. **Focus on graph algorithm visualizations.** Graph algorithms (Dijkstra, BFS/DFS, topological sort, minimum spanning trees) are the hardest to visualize in your head — VisuAlgo's animations are most valuable here.

5. **Don't just watch — implement.** VisuAlgo is a supplement, not a replacement for writing code. After watching the animation, close the website and implement the algorithm yourself.
