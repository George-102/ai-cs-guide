# Reinforcement Learning

## Introduction

- Source: David Silver's Reinforcement Learning (DeepMind), Andrew Ng's RL Specialization, CS234 (Stanford)
- Prerequisites: Machine Learning basics, Python, probability, calculus
- Programming language: Python (Gymnasium, Stable-Baselines3)
- Course difficulty: 🌟🌟🌟🌟
- Estimated study hours: 40+

Reinforcement Learning (RL) is another core branch of AI that studies **how agents learn optimal behavior through interaction with an environment**. Unlike supervised learning, RL doesn't require labeled data — the agent learns through trial and error and reward signals.

AlphaGo defeating the Go world champion, ChatGPT's RLHF (Reinforcement Learning from Human Feedback), robot control, autonomous driving decisions, and your MARL+LLM+RAG air traffic management project — all real-world applications of reinforcement learning.

For you, who wants to build AI Agents and multi-agent systems, reinforcement learning is a **must-master core technology**.

## Core Knowledge Modules

### Fundamental Concepts

- **Markov Decision Process (MDP)**: States, actions, transition probabilities, rewards, discount factor
- **Policy**: The agent's behavior strategy π(a|s)
- **Value functions**: State value V(s), action value Q(s,a)
- **Bellman equations**: Recursive relationships for value functions
- **Exploration vs. Exploitation**: ε-greedy, UCB

### Classic Algorithms

**Value-based algorithms:**
- **Dynamic programming**: Policy iteration, value iteration
- **Monte Carlo methods**: MC prediction, MC control
- **Temporal Difference learning (TD)**: TD(0), TD(λ)
- **Q-Learning**: Off-policy TD control
- **SARSA**: On-policy TD control
- **Deep Q-Network (DQN)**: Q-Learning + deep neural networks

**Policy-based algorithms:**
- **REINFORCE**: Basic policy gradient algorithm
- **Actor-Critic**: Combining value functions and policy gradients
- **A2C/A3C**: Advantage Actor-Critic, asynchronous version
- **PPO (Proximal Policy Optimization)**: The most popular policy gradient algorithm today
- **SAC (Soft Actor-Critic)**: Maximum entropy RL, best for continuous action spaces

**Model-based methods:**
- **Dyna-Q**: Combining model learning and model-free learning
- **MBPO (Model-Based Policy Optimization)**: Model-based + policy optimization

### Deep Reinforcement Learning

- **DQN and its variants**: Double DQN, Dueling DQN, Prioritized Experience Replay
- **Continuous action spaces**: DDPG, TD3, SAC
- **Distributed RL**: Rainbow, R2D2
- **Offline RL**: Learning from fixed datasets without environment interaction

### Multi-Agent Reinforcement Learning (MARL)

- **MARL fundamentals**: Joint action spaces, cooperative vs. competitive vs. mixed
- **Independent learning**: Each agent learns independently
- **Centralized Training with Decentralized Execution (CTDE)**: QMIX, MAPPO
- **Communication and collaboration**: Communication protocols between agents
- **Opponent modeling**: Predicting opponent behavior in competitive environments

### RLHF (Reinforcement Learning from Human Feedback)

- **Reward Model**: Learning reward functions from human preferences
- **PPO + Reward Model**: The core training method behind ChatGPT
- **DPO (Direct Preference Optimization)**: Alternative that doesn't require a reward model
- **Constitutional AI**: Constraining model behavior with rules

## How to Learn

**Start with David Silver, then read Sutton & Barto.** David Silver's course helps you quickly build RL intuition; Sutton & Barto's book helps you deeply understand the theory.

**Start with Gymnasium environments.** Run classic environments like CartPole and MountainCar first to understand the basic RL loop (observe → act → reward → observe).

**Implement classic algorithms from scratch.** Hand-write Q-Learning, REINFORCE, and PPO in NumPy — don't just call libraries. Implementing once beats watching ten times.

**Do CS234 or CS285 labs.** These course labs require you to implement complete RL algorithms and test them in multiple environments — the best hands-on practice.

**Combine with your UAM project.** Use PettingZoo to build multi-agent environments and try QMIX or MAPPO algorithms, applying RL knowledge directly to your air traffic management project.

## Notes

Reinforcement learning is the core technology behind AI Agents and multi-agent systems. Your MARL+LLM+RAG air traffic management project directly depends on RL knowledge.

RL has a higher mathematical bar than deep learning (involving stochastic processes, dynamic programming, optimization theory). Don't be intimidated by formulas — build intuition first, then progressively dive into mathematical derivations.

PPO is the most practical RL algorithm today — prioritize mastering it. SAC performs best in continuous action spaces and is also worth learning.
