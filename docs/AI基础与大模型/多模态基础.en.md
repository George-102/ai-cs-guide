# Multimodal Foundations

## Introduction

- Source: Stanford CS231n (Computer Vision), CLIP paper, HuggingFace multimodal course
- Prerequisites: Python basics, basic understanding of neural network concepts
- Programming language: Python
- Course difficulty: 🌟🌟🌟🌟
- Estimated study hours: 20+

"Modality" refers to the type of information: text is a modality, images are a modality, and so is audio. **Multimodal models** can understand and generate multiple types of information simultaneously — for example, GPT-4V can see images and describe them, GPT-4o can listen to audio and respond with voice, and Sora can generate videos from text.

For CS students, understanding multimodality isn't just about using APIs — it's about knowing the principles behind these capabilities: vision encoders, cross-modal alignment, unified generation architectures. This way, you can make the right technical choices in future projects.

## Notes

You don't need to master every direction in multimodality. The minimum for CS students: understand how vision encoders (ViT) convert images to vectors, and understand how cross-modal alignment (CLIP) puts text and images in the same "semantic space." Once these two concepts are clear, other multimodal models are largely variations on the same theme, and reading papers won't feel overwhelming.
