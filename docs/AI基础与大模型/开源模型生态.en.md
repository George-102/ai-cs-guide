# Open-Source Model Ecosystem

## Introduction

- Source: HuggingFace official documentation and courses, ollama project
- Prerequisites: Python basics, basic understanding of LLM concepts
- Programming language: Python
- Course difficulty: 🌟🌟
- Estimated study hours: 15+

Without understanding the open-source model ecosystem, you can't truly build AI applications. This course teaches you how to find, evaluate, download, and use the right model for your needs — whether via cloud API or running locally.

The current open-source LLM ecosystem consists of several core components: **HuggingFace Hub** (model & dataset hosting platform, like GitHub for AI), **open-source models** (Llama, Qwen, DeepSeek, Mistral, etc.), **local inference tools** (ollama, llama.cpp, vLLM), and **model selection methodology** (how to choose based on your hardware and task).

For CS students, understanding this ecosystem matters because: it keeps you from being locked into any single provider, and lets you flexibly choose solutions based on cost, privacy, latency, and other requirements.

## Resource Summary

**Getting started with HuggingFace:**

- [HuggingFace NLP Course](https://huggingface.co/learn/nlp-course/en/) — Free English course, covers Transformers library usage, model download, fine-tuning, deployment
- [HuggingFace Model Hub](https://huggingface.co/models) — Browse and search 100K+ open-source models
- [HuggingFace Tokenizers](https://huggingface.co/docs/tokenizers/) — Understand how text is "read" by models

**Running models locally:**

- [ollama](https://ollama.com) — One command to run open-source models, supports Llama, Qwen, DeepSeek, Mistral, and more. Best for getting started
- [llama.cpp](https://github.com/ggerganov/llama.cpp) — C++ local inference engine, can run quantized models on laptop CPUs, supports GGUF format
- [LM Studio](https://lmstudio.ai/) — GUI for running models locally, great for those uncomfortable with the command line
- [vLLM](https://github.com/vllm-project/vllm) — High-performance inference engine for high-throughput server-side deployment

**Model selection guide:**

- [HuggingFace Open LLM Leaderboard](https://huggingface.co/spaces/open-llm-leaderboard/open_llm_leaderboard) — Benchmark rankings for open-source models
- [Chatbot Arena](https://chat.lmsys.org/) — Human blind-test voting rankings, reflecting real-world experience
- [Can I run this model?](https://huggingface.co/spaces/Vokturz/can-it-run-llm) — Check whether your hardware can run a given model

**Chinese open-source models (2025 mainstream):**

- [Qwen (Tongyi Qianwen)](https://huggingface.co/Qwen) — Alibaba open-source, strong Chinese capabilities, full range (0.5B-72B)
- [DeepSeek](https://huggingface.co/deepseek-ai) — DeepSeek-V3/R1, performance approaching top closed-source models
- [GLM (Zhipu)](https://huggingface.co/THUDM) — Tsinghua open-source, strong bilingual Chinese-English capabilities
- [MiniCPM](https://huggingface.co/openbmb) — BAAI open-source, star among small on-device models

## Notes

Recommended path: install ollama and pull a 7B model first (get a feel for it) → browse HuggingFace Hub for interesting models → learn to write inference scripts with the Transformers library → pick a direction to go deeper (local optimization or cloud deployment) based on your needs. Don't try to understand all tools at once — start using them first.
