# Deep Learning

## Introduction

- Source: Andrew Ng's Deep Learning Specialization (Coursera), Hung-Yi Lee's Deep Learning, CS231n (Stanford)
- Prerequisites: Machine Learning basics, Python, linear algebra, calculus, probability
- Programming language: Python (PyTorch primary, TensorFlow secondary)
- Course difficulty: 🌟🌟🌟🌟
- Estimated study hours: 50+

Deep Learning is the engine of the current AI revolution. From GPT to Stable Diffusion, from AlphaFold to autonomous driving — these breakthroughs are all built on deep learning. Deep learning is essentially a branch of machine learning that uses multi-layer neural networks to learn hierarchical representations of data.

If machine learning is the foundation of AI, then deep learning is AI's **core weapon**. Understanding deep learning is essential to truly understanding large language models, multimodal AI, and generative AI at a fundamental level.

## Core Knowledge Modules

### Neural Network Fundamentals

- **Perceptron and Multi-Layer Perceptron (MLP)**
- **Backpropagation**: Chain rule, computation graphs
- **Activation functions**: ReLU, Sigmoid, Tanh, GELU, SwiGLU
- **Loss functions**: Cross-entropy, MSE, Hinge Loss
- **Optimization algorithms**: SGD, Momentum, Adam, AdamW
- **Regularization**: Dropout, Batch Normalization, Weight Decay
- **Weight initialization**: Xavier, He initialization

### Convolutional Neural Networks (CNN)

- **Convolution operation**: Kernels, stride, padding
- **Pooling**: Max Pooling, Average Pooling
- **Classic architectures**: LeNet, AlexNet, VGG, GoogLeNet, ResNet
- **Modern architectures**: EfficientNet, ConvNeXt
- **Applications**: Image classification, object detection, image segmentation

### Recurrent Neural Networks (RNN)

- **RNN basics**: Time unrolling, vanishing/exploding gradients
- **LSTM**: Gating mechanisms, forget gate/input gate/output gate
- **GRU**: Simplified LSTM
- **Applications**: Sequence modeling, time series prediction

### Transformer Architecture

- **Self-Attention mechanism**: Query, Key, Value
- **Multi-Head Attention**
- **Positional encoding**: Sinusoidal, RoPE
- **Transformer encoder-decoder architecture**
- **BERT**: Pre-training + fine-tuning paradigm
- **GPT series**: Autoregressive language models
- **ViT**: Vision Transformer, applying Transformers to vision

### Generative Models

- **Autoencoders (AE) and Variational Autoencoders (VAE)**
- **Generative Adversarial Networks (GAN)**: Generator, discriminator, training tricks
- **Diffusion Models**: DDPM, Stable Diffusion
- **Flow-based models**

### Training Techniques

- **Learning rate scheduling**: Warm-up, Cosine Decay, ReduceLROnPlateau
- **Data augmentation**: Flip, crop, Mixup, CutMix
- **Mixed precision training**: FP16/BF16
- **Distributed training**: Data parallelism, model parallelism, pipeline parallelism
- **Gradient accumulation**: Simulating larger batch sizes

## How to Learn

**Start with Andrew Ng, then read d2l.** Andrew Ng's course builds the overall framework; d2l.ai helps you implement each algorithm in PyTorch. Together, they cover both theory and practice.

**Implement every model from scratch with PyTorch.** Don't just call high-level APIs. Write MLP, CNN, RNN, and Transformer from scratch — you'll gain a completely different understanding of these architectures.

**Do the CS231n labs.** These labs require you to implement CNNs from scratch with NumPy — the best hands-on practice for deep learning beginners.

**Reproduce classic papers.** Pick a classic paper (e.g., ResNet, Transformer) and try to reproduce its core experiments. This is the key step from "can use" to "master."

## Notes

Deep learning is the foundation for understanding large language models, multimodal AI, and generative AI. Even if you primarily use off-the-shelf models and APIs today, understanding the underlying principles helps you make better judgments when problems arise.

PyTorch is the dominant framework in both academia and industry — prioritize learning it. TensorFlow still has significant enterprise deployment use and can be learned as a supplement.
