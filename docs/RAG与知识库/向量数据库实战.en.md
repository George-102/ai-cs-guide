# Vector Database in Practice

## Introduction

- Source: [Qdrant](https://qdrant.tech/documentation/)、[Chroma](https://docs.trychroma.com/)、[Pinecone](https://docs.pinecone.io/)
- Prerequisites: Python basics, understanding of Embedding concepts
- Programming language: Python
- Course difficulty: 🌟🌟
- Estimated study hours: 10+

Vector databases are the "memory" component of RAG systems. Traditional databases store structured data (rows and columns); vector databases store **vectors** (Embeddings) — numerical semantic representations of text, images, and other content. Their core capability: **given a query, quickly find the most semantically similar vectors.**

This capability is called "Approximate Nearest Neighbor search" (ANN) — the underlying engine that lets RAG systems quickly retrieve relevant knowledge. Without a vector database, RAG would need to brute-force scan all documents — completely impractical at scale.

## Notes

Recommended path: **Chroma (local development) → Qdrant (production deployment)**. Chroma requires no services to install — just pip install and go, perfect for learning. Qdrant single-node edition starts with one Docker command, feature-complete and a solid production choice.

pgvector is also worth watching — if your project already uses PostgreSQL, adding one extension gives you vector search without introducing any new components.

Three key concepts to understand: **Embedding** (text → vector), **ANN search** (quickly finding similar vectors), and **metadata filtering** (vector similarity + structured condition hybrid queries).
