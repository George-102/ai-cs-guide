# Computer Networks

## What Is Computer Networks

Computer networks study **how computers reliably exchange data with each other**. From opening a browser to visit a webpage, to sending a message on WeChat, to communication between nodes in globally distributed systems — all of this depends on computer networks.

The entire internet runs on a layered protocol architecture: the physical layer transmits electrical signals, the link layer handles frame transmission between adjacent nodes, the network layer handles routing across multiple networks (IP), the transport layer handles end-to-end reliable delivery (TCP/UDP), and the application layer is what you directly interact with (HTTP, DNS, SMTP).

## Why Study Computer Networks

**Foundation of all distributed systems.** Microservice architecture, cloud computing, distributed databases, edge computing — all of these technologies are built on top of networks. Without understanding network latency, packet loss, and congestion control, you cannot design reliable distributed systems.

**Interview essential.** "TCP three-way handshake," "difference between HTTP and HTTPS," "DNS resolution process" — these questions appear in virtually every technical interview.

**Understanding real-world communication.** Your UAM air traffic management project involves multi-agent communication. 5G network slicing and V2X communication protocols are built on networking knowledge.

**Core subject in CS entrance exams.** Tests the OSI/TCP model, protocols at each layer, routing algorithms, and more.

## Core Knowledge Modules

### Network Architecture

- **OSI 7-layer model** vs. **TCP/IP 4-layer model**
- Responsibilities and representative protocols of each layer
- Encapsulation and decapsulation: how data flows from application layer to physical layer

### Physical and Link Layers

- Transmission media: twisted pair, fiber optics, wireless
- Encoding and modulation
- Ethernet frame structure, MAC addresses
- ARP protocol: IP address to MAC address mapping
- How switches work

### Network Layer

- **IP protocol**: IPv4 packet format, subnetting, CIDR
- **Routing algorithms**: Distance Vector (RIP), Link State (OSPF), Path Vector (BGP)
- **ICMP**: How Ping and Traceroute work
- **NAT**: Network address translation, its role and limitations
- IPv6 fundamentals

### Transport Layer

- **TCP**: Three-way handshake, four-way teardown, sliding window, congestion control (slow start, fast retransmit, fast recovery)
- **UDP**: Connectionless, unreliable but efficient
- TCP vs. UDP: when to use which
- Port numbers and sockets

### Application Layer

- **HTTP/HTTPS**: Request methods, status codes, headers, cookies/sessions
- **DNS**: Domain resolution process, recursive vs. iterative queries
- **SMTP/POP3/IMAP**: Email protocols
- **WebSocket**: Full-duplex communication
- **REST API** design basics

## How to Learn

**Top-down, starting from what you already know.** You use HTTP and DNS every day, so start there for the quickest wins. Recommended book: *Computer Networking: A Top-Down Approach* — it starts from the application layer and works down, perfect for self-study.

**Get hands-on with experiments.** Use Wireshark to capture packets and see what real TCP handshakes and HTTP requests look like. Seeing real packets teaches more than reading theory ten times over.

**Do the Stanford CS144 labs.** This course has you implement a TCP/IP protocol stack. After completing these labs, your understanding of networking will surpass 90% of CS graduates.

**Draw state transition diagrams.** TCP state transitions (LISTEN, SYN_SENT, ESTABLISHED, TIME_WAIT...) are a frequent interview topic. Drawing them out is far more effective than memorizing tables.

## Recommended Resources

**Textbooks**

- *Computer Networking: A Top-Down Approach* (Kurose & Ross) — Recommended as primary textbook, starts from application layer, great for self-study
- *Computer Networks* (Tanenbaum) — Classic textbook with deep treatment of networking concepts
- *TCP/IP Illustrated, Volume 1* (W. Richard Stevens) — Deep dive into the TCP/IP protocol suite

**Courses**

- [Stanford CS144: Introduction to Networking](https://cs144.github.io/) — Lab-driven, implement the protocol stack, highly recommended
- [Harvard CS148: Introduction to Computer Networking](https://cs148.net/) — Another high-quality course
- [UMich EECS 489: Computer Networks](https://eecs489.org/) — University of Michigan's networking course

**Practical Tools**

- [Wireshark](https://www.wireshark.org/) — Network packet analyzer, essential
- [tcpdump](https://www.tcpdump.org/) — Command-line packet capture tool
- [netcat (nc)](https://nc110.sourceforge.io/) — Networking Swiss Army knife
- [Postman](https://www.postman.com/) — HTTP API debugging tool
