# 计算机网络

## 什么是计算机网络

计算机网络研究的是**计算机之间如何可靠地交换数据**。从你打开浏览器访问一个网页，到微信发送一条消息，再到全球分布式系统的节点间通信——所有这些都依赖计算机网络。

整个互联网的运行建立在分层协议体系之上：物理层负责传输电信号，链路层负责相邻节点间的帧传输，网络层负责跨越多个网络的路由（IP），传输层负责端到端可靠交付（TCP/UDP），应用层才是你直接接触的部分（HTTP、DNS、SMTP）。

## 为什么要学计算机网络

**一切分布式系统的基础。** 微服务架构、云计算、分布式数据库、边缘计算——所有这些技术的底层都是网络。不理解网络延迟、丢包、拥塞控制，你无法设计出可靠的分布式系统。

**面试硬通货。** "TCP 三次握手"、"HTTP 和 HTTPS 的区别"、"DNS 解析过程"——这些问题在几乎所有技术面试中都会出现。

**理解现实世界的通信。** 你做的 UAM 空中交通管理项目涉及多机通信，5G 网络切片、V2X 通信协议都建立在网络知识之上。

**考研 408 核心科目。** 分值约 25 分，重点考察 OSI/TCP 模型、各层协议、路由算法等。

## 核心知识模块

### 网络体系结构

- **OSI 七层模型** vs **TCP/IP 四层模型**
- 每层的职责和代表性协议
- 封装与解封装：数据从应用层到物理层的过程

### 物理层与链路层

- 传输介质：双绞线、光纤、无线
- 编码与调制
- 以太网帧结构、MAC 地址
- ARP 协议：IP 地址到 MAC 地址的映射
- 交换机的工作原理

### 网络层

- **IP 协议**：IPv4 报文格式、子网划分、CIDR
- **路由算法**：距离向量（RIP）、链路状态（OSPF）、路径向量（BGP）
- **ICMP**：Ping 和 Traceroute 的原理
- **NAT**：网络地址转换的作用与局限
- IPv6 基本概念

### 传输层

- **TCP**：三次握手、四次挥手、滑动窗口、拥塞控制（慢启动、快重传、快恢复）
- **UDP**：无连接、不可靠但高效
- TCP vs UDP 的适用场景
- 端口号与套接字（Socket）

### 应用层

- **HTTP/HTTPS**：请求方法、状态码、Header、Cookie/Session
- **DNS**：域名解析过程、递归查询 vs 迭代查询
- **SMTP/POP3/IMAP**：邮件协议
- **WebSocket**：全双工通信
- **REST API** 设计基础

### 网络安全（基础）

- TLS/SSL 握手过程
- 对称加密 vs 非对称加密
- 数字证书与 CA
- 常见攻击：DDoS、中间人攻击、DNS 劫持

## 怎么学

**自顶向下，从你熟悉的部分开始。** 你每天都在用 HTTP 和 DNS，从这里入手最有成就感。推荐《计算机网络：自顶向下方法》——它从应用层讲起，逐步深入到底层，非常适合自学。

**动手做实验。** 用 Wireshark 抓包，看看真实的 TCP 握手、HTTP 请求长什么样。理论看十遍不如抓一次包。

**做 Stanford CS144 实验。** 这门课要求你实现一个 TCP/IP 协议栈。写完这些实验，你对网络的理解会超过 90% 的 CS 毕业生。

**画状态转换图。** TCP 的状态转换（LISTEN、SYN_SENT、ESTABLISHED、TIME_WAIT...）是面试高频考点，画出来比背表格好记得多。

## 推荐资源

**教材**

- 《计算机网络：自顶向下方法》（Kurose & Ross）— 推荐作为主要教材，从应用层开始讲，适合自学
- 《计算机网络》（谢希仁）— 国内经典教材，适合考研复习
- 《TCP/IP 详解（卷 1）》（W. Richard Stevens）— 深入理解 TCP/IP 协议族

**课程**

- [Stanford CS144: Introduction to Networking](https://cs144.github.io/) — 实验驱动，实现协议栈，强烈推荐
- [中科大郑烇《计算机网络》](https://www.bilibili.com/video/BV1JV411t7ow) — 中文讲解清晰，适合系统学习
- [Harvard CS148: Introduction to Computer Networking](https://cs148.net/) — 另一门高质量的英文课程

**实践工具**

- [Wireshark](https://www.wireshark.org/) — 网络抓包分析工具，必备
- [tcpdump](https://www.tcpdump.org/) — 命令行抓包工具
- [netcat (nc)](https://nc110.sourceforge.io/) — 网络调试瑞士军刀
- [Postman](https://www.postman.com/) — HTTP API 调试工具
