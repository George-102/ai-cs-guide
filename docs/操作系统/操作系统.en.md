# Operating Systems

## What is an Operating System

An Operating System (OS) is system software that manages computer hardware and software resources. It acts as an intermediary between applications and hardware, responsible for allowing multiple programs to share CPU, memory, disk, and network resources without each program needing to manage these low-level details on its own.

You interact with an operating system every day — Windows, Linux, and macOS are all operating systems. When you click a program icon, the OS loads it into memory, allocates CPU time, and handles file I/O. When you open a browser, the OS manages the network connection. Behind these seemingly simple tasks, the OS is coordinating dozens or even hundreds of concurrent tasks.

## Why Learn Operating Systems

**Understand what your programs actually do.** When you call `printf`, it involves system calls, transitions between user mode and kernel mode, buffer management, and device drivers. Without understanding the OS, your programs are black boxes — they run, but you don't know why, and you can't troubleshoot when things go wrong.

**Write more efficient programs.** Understanding process scheduling, memory paging, and caching mechanisms helps you write system-friendly code. For example, knowing disk I/O is a bottleneck motivates you to reduce unnecessary file operations; knowing the memory page size is 4KB helps you better organize data structures.

**Essential for interviews and exams.** Operating systems is a core subject for graduate school entrance exams and technical interviews. Questions about the difference between processes and threads, conditions for deadlocks, and virtual memory principles appear with high frequency.

**Foundation for advanced topics.** Computer networking, databases, and distributed systems all build on OS concepts. Without a solid OS foundation, you'll struggle with these subjects later.

## Core Knowledge Areas

### Processes and Threads

- A **process** is an instance of a program execution with its own address space
- A **thread** is an execution unit within a process; threads in the same process share the address space
- Inter-Process Communication (IPC): pipes, message queues, shared memory, semaphores
- Thread synchronization: mutexes, condition variables, read-write locks

### Memory Management

- **Virtual memory**: each process has its own virtual address space; the OS maps it to physical memory
- **Paging and segmentation**: two fundamental approaches to memory management
- **Page replacement algorithms**: LRU, FIFO, Clock — decide which pages to evict when memory is full
- **Memory allocation**: difference between heap and stack, how malloc works under the hood

### File Systems

- File organization: directory tree structure
- Disk scheduling algorithms: SCAN, C-SCAN, SSTF
- File system implementation: inodes, journaling file systems

### I/O and Device Management

- I/O models: blocking, non-blocking, multiplexing (select/poll/epoll)
- The role of device drivers
- Interrupt handling mechanisms

### Concurrency and Synchronization

- The critical section problem
- Classic synchronization problems: producer-consumer, readers-writers, dining philosophers
- Four necessary conditions for deadlock and prevention strategies

## How to Learn

**Build the big picture first.** Don't dive into details right away. Start by understanding the OS's core responsibilities (managing CPU, memory, disk, devices), then tackle each area one by one.

**Learn by doing experiments.** OS is a highly practical course. Reading without coding leaves concepts on paper. The xv6 labs from MIT are highly recommended — a teaching OS with a small codebase and clear structure, perfect for beginners.

**Draw diagrams to aid understanding.** Process state transitions, memory address mapping, disk scheduling — these concepts are abstract in text but clear when visualized.

**Don't skip Linux commands.** Learning OS isn't purely theoretical. Tools like `ps`, `top`, `strace`, and `lsof` let you observe OS behavior in action.

## Recommended Resources

**Textbooks**

- [Operating Systems: Three Easy Pieces](https://pages.cs.wisc.edu/~remzi/OSTEP/) — Free online textbook, clearly written, great for beginners
- *Operating System Concepts* (the Dinosaur Book) — Classic textbook, comprehensive coverage
- *Modern Operating Systems* (Andrew Tanenbaum) — Another classic, focuses on design principles

**Courses**

- [MIT 6.S081](https://pdos.csail.mit.edu/6.828/2023/) — MIT's OS course based on xv6, with complete labs
- [UC Berkeley CS162](https://cs162.org/) — Berkeley's OS course with videos and assignments

**Practice**

- [xv6](https://pdos.csail.mit.edu/6.828/2023/xv6.html) — MIT's teaching OS, approximately 10,000 lines of code
- [Write your own OS](https://github.com/cfenollosa/os-tutorial) — A tutorial for building an OS from scratch
