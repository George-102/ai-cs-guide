# Computer Architecture

## What is Computer Architecture

Computer Architecture studies the organization of computer systems — how CPUs execute instructions, how memory stores and retrieves data, and how various components work together. The core question it answers is: **how does a computer run the programs you write?**

Your Python code ultimately becomes machine instructions that the CPU can execute. From high-level languages to assembly to machine code, from CPU pipelines to cache hierarchies to memory buses — architecture studies the design and optimization of every layer in this chain.

## Why Learn Computer Architecture

**Understand why programs are fast or slow.** The same algorithm implemented with arrays is much faster than with linked lists — the reason is cache locality. Without understanding architecture, you can only guess based on experience; with it, you can make informed optimization decisions.

**Understand "abstraction leaks."** Programming languages provide many abstractions, but they aren't always reliable. Integer overflow, floating-point precision, concurrency races — the root causes of these problems are at the hardware level. Understanding architecture lets you see through abstractions to what's actually happening underneath.

**Foundation for embedded and systems programming.** If you pursue embedded development, device drivers, compilers, or performance optimization, architecture is an essential prerequisite.

**Key topic for exams and interviews.** Instruction pipelines, cache principles, and bus arbitration are core topics in the 408 graduate entrance exam and common interview questions for systems-related positions.

## Core Knowledge Areas

### Data Representation

- Binary and hexadecimal
- Integer representation: sign-magnitude, one's complement, two's complement
- Floating-point representation: IEEE 754 standard
- Character encoding: ASCII, UTF-8

### Instruction Set Architecture

- ISA: CISC vs RISC
- Common instruction types: arithmetic, logical, control flow, memory access
- Assembly language basics: x86 or RISC-V

### CPU Design

- Datapath: ALU, registers, buses
- Control unit: how instructions become control signals
- Instruction pipelining: fetch, decode, execute, memory access, write-back
- Pipeline hazards: data hazards, control hazards, structural hazards

### Memory Hierarchy

- **Cache**: how L1/L2/L3 caches work, mapping policies, replacement strategies
- **Main memory (DRAM)**: memory organization, bandwidth, latency
- **Virtual memory**: address translation, TLB, page replacement
- **Memory locality**: temporal locality and spatial locality

### Buses and I/O

- Bus types: data bus, address bus, control bus
- I/O methods: programmed I/O, interrupts, DMA
- Common interfaces: PCIe, USB, SATA

### Parallel Processing

- Instruction-Level Parallelism (ILP): superscalar, out-of-order execution
- Data-Level Parallelism: SIMD, GPU
- Thread-Level Parallelism: multi-core processors, shared memory

## How to Learn

**Start with C and assembly.** Many architecture concepts are best understood through code. Write simple C programs and use `gcc -S` to see the generated assembly code, building intuition from high-level languages to hardware.

**Use simulation tools.** Use [Ripes](https://github.com/mortbopet/Ripes) or [Venus](https://venus.cs61c.org/) to simulate RISC-V processor execution and observe data flow at each step.

**Draw datapath diagrams.** The CPU datapath is the core of architecture. Drawing how data flows between components during instruction execution is more effective than reading the textbook ten times.

**Compare different processors.** Understand the design differences between x86 and RISC-V, and the trade-offs between CISC and RISC philosophies.

## Recommended Resources

**Textbooks**

- *Computer Organization and Design: The Hardware/Software Interface* (Patterson & Hennessy) — Classic textbook, the RISC-V edition is great for beginners
- *Computer Architecture: A Quantitative Approach* (Hennessy & Patterson) — Advanced classic for deeper study
- *Computer Systems: A Programmer's Perspective* (CS:APP) — Architecture from a programmer's view, highly practical

**Courses**

- [CS61C (UC Berkeley)](https://cs61c.org/) — Berkeley's architecture course with complete RISC-V labs
- [Nand2Tetris](https://www.nand2tetris.org/) — Build a complete computer system step by step, starting from NAND gates
- [CMU 15-213](http://www.cs.cmu.edu/~213/) — CS:APP companion course, covers systems-level programming

**Practice**

- [Ripes](https://github.com/mortbopet/Ripes) — RISC-V processor simulator with pipeline visualization
- [Nand2Tetris Projects](https://www.nand2tetris.org/project) — Complete hands-on practice from logic gates to an operating system
- [godbolt.org](https://godbolt.org/) — Online compiler to see assembly output from different compilers
