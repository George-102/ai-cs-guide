# 计算机体系结构

## 什么是计算机体系结构

计算机体系结构（Computer Architecture）研究的是计算机系统的组织方式 — CPU 怎么执行指令、内存怎么存取数据、各个部件怎么协同工作。它回答的核心问题是：**计算机是怎么运行你写的程序的？**

你写的 Python 代码最终要变成 CPU 能执行的机器指令。从高级语言到汇编到机器码，从 CPU 的流水线到缓存层次到内存总线 — 体系结构就是研究这条链路上每一层的设计和优化。

## 为什么要学计算机体系结构

**知道程序为什么快或者慢。** 同样的算法，用数组实现比链表快得多 — 原因是缓存局部性（cache locality）。不理解体系结构，你只能靠经验猜；理解了，你就能做出有依据的优化决策。

**理解"抽象泄漏"。** 编程语言提供了很多抽象，但这些抽象不总是可靠的。整数溢出、浮点精度、并发竞争 — 这些问题的根源都在硬件层面。理解体系结构，你就能穿透抽象，看到底层到底发生了什么。

**为嵌入式和系统编程打基础。** 如果你将来做嵌入式开发、驱动程序、编译器、性能优化，体系结构是绕不开的必修课。

**面试和考研的重点。** 指令流水线、Cache 原理、总线仲裁 — 这些是 408 考研的核心考点，也是大厂系统类岗位的常见面试题。

## 核心知识模块

### 数据表示

- 二进制、十六进制
- 整数表示：原码、反码、补码
- 浮点数表示：IEEE 754 标准
- 字符编码：ASCII、UTF-8

### 指令系统

- 指令集架构（ISA）：CISC vs RISC
- 常见指令类型：算术、逻辑、控制流、访存
- 汇编语言基础：x86 或 RISC-V

### CPU 设计

- 数据通路（Datapath）：ALU、寄存器、总线
- 控制单元：怎么从指令到控制信号
- 指令流水线：取指、译码、执行、访存、写回
- 流水线冒险：数据冒险、控制冒险、结构冒险

### 存储层次

- **缓存（Cache）**：L1/L2/L3 缓存的工作原理、映射方式、替换策略
- **主存（DRAM）**：内存组织、带宽、延迟
- **虚拟内存**：地址转换、TLB、页面置换
- **存储局部性**：时间局部性和空间局部性

### 总线与 I/O

- 总线类型：数据总线、地址总线、控制总线
- I/O 方式：程序查询、中断、DMA
- 常见接口：PCIe、USB、SATA

### 并行处理

- 指令级并行（ILP）：超标量、乱序执行
- 数据级并行：SIMD、GPU
- 线程级并行：多核处理器、共享内存

## 怎么学

**从 C 语言和汇编入手。** 体系结构的很多概念需要通过代码来理解。写一些简单的 C 程序，用 `gcc -S` 看生成的汇编代码，建立从高级语言到硬件的直觉。

**做模拟实验。** 用 [Ripes](https://github.com/mortbopet/Ripes) 或 [Venus](https://venus.cs61c.org/) 模拟 RISC-V 处理器的运行过程，观察每一步的数据流动。

**画数据通路图。** CPU 的数据通路是体系结构的核心。画出指令执行过程中数据在各个部件之间的流动，比看十遍书都管用。

**对比不同处理器。** 了解 x86 和 RISC-V 的设计差异，理解 CISC 和 RISC 两种设计哲学的取舍。

## 推荐资源

**教材**

- 《计算机组成与设计：硬件/软件接口》（Patterson & Hennessy）— 经典教材，RISC-V 版本适合入门
- 《计算机体系结构：量化研究方法》（Hennessy & Patterson）— 进阶经典，适合深入学习
- 《深入理解计算机系统》（CS:APP）— 从程序员视角看体系结构，实践性极强

**课程**

- [CS61C (UC Berkeley)](https://cs61c.org/) — Berkeley 的体系结构课程，有完整的 RISC-V 实验
- [Nand2Tetris](https://www.nand2tetris.org/) — 从与非门开始，一步步构建完整的计算机系统
- [CMU 15-213](http://www.cs.cmu.edu/~213/) — CS:APP 配套课程，讲解系统级编程

**实践**

- [Ripes](https://github.com/mortbopet/Ripes) — RISC-V 处理器模拟器，可视化流水线运行
- [Nand2Tetris 项目](https://www.nand2tetris.org/project) — 从逻辑门到操作系统的完整实践
- [godbolt.org](https://godbolt.org/) — 在线编译器，查看不同编译器生成的汇编代码
