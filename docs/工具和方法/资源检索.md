# 资源检索指南

## 为什么需要这门课

信息过载是 AI 时代最大的学习障碍之一。网上有海量的课程、教程、论文和工具，但质量参差不齐。**找到好资源的能力，本身就是一种核心竞争力。**

吴恩达在 Buildathon 上提到：AI 工程师最重要的技能之一就是"快速找到正确的信息"。这不仅仅是 Google 搜索那么简单——你需要知道去哪里搜、怎么搜、如何判断资源的质量。

## 核心搜索策略

### 明确你的搜索目标

在搜索之前，先问自己三个问题：
1. **我要解决什么具体问题？**（而不是"我想学 AI"这样模糊的目标）
2. **我目前的理解水平是什么？**（入门、进阶、深入？）
3. **我需要什么类型的资源？**（视频、文档、论文、代码示例？）

### 搜索渠道分层

**第一层：搜索引擎**
- Google（需要科学上网）：最全面的搜索，用英文关键词效果更好
- 百度：中文内容搜索，但要注意筛选质量
- Bing：介于两者之间，某些技术问题搜索结果不错

**第二层：专业平台**
- [GitHub](https://github.com/)：搜索项目、代码示例、awesome-list
- [arXiv](https://arxiv.org/)：最新学术论文
- [Google Scholar](https://scholar.google.com/)：学术论文搜索
- [Semantic Scholar](https://www.semanticscholar.org/)：AI 驱动的论文搜索
- [Papers With Code](https://paperswithcode.com/)：论文 + 代码实现

**第三层：社区和问答**
- [Stack Overflow](https://stackoverflow.com/)：编程问题首选
- [Hacker News](https://news.ycombinator.com/)：技术新闻和深度讨论
- [Reddit](https://www.reddit.com/)：r/MachineLearning、r/LocalLLaMA 等子版块
- [知乎](https://www.zhihu.com/)：中文技术社区
- [V2EX](https://www.v2ex.com/)：中文开发者社区

**第四层：AI 辅助搜索**
- [Perplexity](https://www.perplexity.ai/)：AI 搜索引擎，自动汇总多个来源
- [Phind](https://www.phind.com/)：面向开发者的 AI 搜索
- [Devv](https://devv.ai/)：中文 AI 编程搜索

### 搜索技巧

**用英文搜索技术问题。** 英文技术社区的讨论质量和深度远超中文。即使你的英文不够好，用翻译软件把中文问题翻译成英文再搜，结果也会好很多。

**善用搜索操作符：**
- `site:github.com` — 只在 GitHub 搜索
- `site:arxiv.org` — 只在 arXiv 搜索
- `"exact phrase"` — 精确匹配
- `intitle:关键词` — 标题中包含关键词
- `filetype:pdf` — 只搜索 PDF 文件

**搜索课程资源的关键词模板：**
- `[课程名] + syllabus` — 找课程大纲
- `[课程名] + lecture notes` — 找课程笔记
- `[课程名] + assignments` — 找课程作业
- `[课程名] + projects` — 找课程项目
- `[课程名] + best resources` — 找推荐资源汇总
- `awesome-[topic]` — 找 GitHub 上的资源汇总列表

## 如何判断资源质量

### 课程资源

- **看来源**：顶尖大学（MIT、Stanford、CMU、Berkeley）的课程质量最有保障
- **看更新频率**：技术课程需要关注是否最近 2-3 年内更新过
- **看是否有实验/项目**：好课程一定有动手实践环节
- **看评价**：Coursera/edX 上的评分和评论有参考价值

### 技术文章

- **看作者**：知名研究者、工程师或团队的文章更可靠
- **看日期**：AI 领域变化极快，2020 年前的文章可能已经过时
- **看是否有代码**：有代码实现的文章比纯理论文章更有参考价值
- **看引用量**：被其他文章引用的次数反映了影响力

### 论文

- **看发表 venue**：顶会（NeurIPS、ICML、ICLR、ACL、CVPR）质量有保障
- **看引用量**：高引用量通常意味着影响力大
- **看是否有代码**：Papers With Code 上能查到实现
- **先看摘要和图**：5 分钟内判断论文是否值得深读

## 建立你的个人知识库

找到好资源只是第一步，**整理和消化**才是关键：

1. **用 Obsidian 做笔记**：把学到的知识用自己的话写下来，建立知识之间的关联
2. **用 GitHub 管理代码**：把练习代码和项目放到 GitHub，既是备份也是作品集
3. **定期回顾**：每周花 30 分钟回顾本周学到的内容，加深记忆
4. **输出倒逼输入**：写博客、做分享、回答问题——教别人是最好的学习方式
