# Guide to Resource Discovery

## Why You Need This Guide

Information overload is one of the greatest learning obstacles in the AI era. The internet is flooded with courses, tutorials, papers, and tools of wildly varying quality. **The ability to find good resources is itself a core competency.**

As Andrew Ng mentioned at Buildathon: one of the most important skills for AI engineers is "quickly finding the right information." This goes far beyond simple Google searches — you need to know where to search, how to search, and how to judge resource quality.

## Core Search Strategies

### Define Your Search Goal

Before searching, ask yourself three questions:
1. **What specific problem am I trying to solve?** (Not vague goals like "I want to learn AI")
2. **What is my current level of understanding?** (Beginner, intermediate, advanced?)
3. **What type of resource do I need?** (Video, documentation, paper, code example?)

### Search Channel Layers

**Layer 1: Search engines**
- Google (requires proxy): Most comprehensive; English keywords work better
- Baidu: Chinese content, but quality filtering is essential
- Bing: A middle ground; decent results for some technical topics

**Layer 2: Specialized platforms**
- [GitHub](https://github.com/): Search projects, code examples, awesome-lists
- [arXiv](https://arxiv.org/): Latest academic papers
- [Google Scholar](https://scholar.google.com/): Academic paper search
- [Semantic Scholar](https://www.semanticscholar.org/): AI-powered paper search
- [Papers With Code](https://paperswithcode.com/): Papers + code implementations

**Layer 3: Communities and Q&A**
- [Stack Overflow](https://stackoverflow.com/): First choice for programming questions
- [Hacker News](https://news.ycombinator.com/): Tech news and deep discussions
- [Reddit](https://www.reddit.com/): Subreddits like r/MachineLearning, r/LocalLLaMA
- [Hacker News](https://news.ycombinator.com/): Tech news and in-depth discussion

**Layer 4: AI-assisted search**
- [Perplexity](https://www.perplexity.ai/): AI search engine that aggregates multiple sources
- [Phind](https://www.phind.com/): AI search for developers
- [Devv](https://devv.ai/): Chinese AI programming search

### Search Techniques

**Search in English for technical questions.** English-language technical communities have far higher quality discussions and depth than Chinese ones. Even if your English isn't great, translate your Chinese question into English before searching — the results will be significantly better.

**Use search operators:**
- `site:github.com` — Search only GitHub
- `site:arxiv.org` — Search only arXiv
- `"exact phrase"` — Exact match
- `intitle:keyword` — Keyword in title
- `filetype:pdf` — PDF files only

**Keyword templates for finding courses:**
- `[course name] + syllabus` — Find course syllabi
- `[course name] + lecture notes` — Find lecture notes
- `[course name] + assignments` — Find assignments
- `[course name] + projects` — Find projects
- `[course name] + best resources` — Find curated resource lists
- `awesome-[topic]` — Find GitHub awesome-lists

## How to Judge Resource Quality

### Course Resources

- **Check the source**: Courses from top universities (MIT, Stanford, CMU, Berkeley) have the highest quality assurance
- **Check update frequency**: Tech courses should have been updated within the last 2-3 years
- **Check for labs/projects**: Good courses always have hands-on components
- **Check reviews**: Ratings on Coursera/edX are useful references

### Technical Articles

- **Check the author**: Articles from well-known researchers, engineers, or teams are more reliable
- **Check the date**: AI moves extremely fast; pre-2020 articles may be outdated
- **Check for code**: Articles with code implementations are more valuable than pure theory
- **Check citations**: Number of citations reflects influence

### Papers

- **Check the venue**: Top conferences (NeurIPS, ICML, ICLR, ACL, CVPR) have quality guarantees
- **Check citation count**: High citations usually means high impact
- **Check for code**: Look up implementations on Papers With Code
- **Read the abstract and figures first**: Decide within 5 minutes whether a paper is worth deep reading

## Build Your Personal Knowledge Base

Finding good resources is only the first step — **organizing and internalizing** is what matters:

1. **Take notes in Obsidian**: Write what you learn in your own words, build connections between concepts
2. **Manage code on GitHub**: Store practice code and projects on GitHub — it's both backup and portfolio
3. **Review regularly**: Spend 30 minutes each week reviewing what you learned to reinforce memory
4. **Output drives input**: Write blogs, give talks, answer questions — teaching others is the best way to learn
