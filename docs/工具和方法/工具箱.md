# 工具箱

这里收录一些提升开发效率的实用工具。每个工具都很小，但用上之后就回不去了。

## 包管理器

### [Scoop](https://scoop.sh/)

Windows 下的命令行包管理器。一行命令装软件，自动处理依赖和环境变量，不用再去官网下载安装包、手动配 PATH。

```bash
# 安装软件
scoop install git
scoop install python

# 更新所有软件
scoop update *

# 搜索软件
scoop search nodejs
```

比起 Chocolatey，Scoop 更轻量，默认装在用户目录下，不需要管理员权限，适合开发者日常使用。

### [winget](https://github.com/microsoft/winget-cli)

微软官方的 Windows 包管理器。和 Scoop 互补 — winget 的软件库更偏向桌面应用（浏览器、通讯工具等），Scoop 更偏向开发工具。两个一起用，覆盖大部分需求。

```bash
winget install Google.Chrome
winget install Microsoft.VisualStudioCode
```

## 终端增强

### [lsd](https://github.com/lsd-rs/lsd)

`ls` 命令的现代替代品。默认带图标、颜色、树形展示，输出比原生 `ls` 好看太多。

```bash
# 基本用法和 ls 一样
lsd
lsd -la
lsd --tree
```

建议设为 `ls` 的别名，日常使用无感知替换。

### [zoxide](https://github.com/ajeetdsouza/zoxide)

`cd` 命令的智能替代品。它会记住你访问过的目录，之后只需要输入部分路径就能跳转。

```bash
# 跳转到包含 "project" 的目录
z project

# 跳转到包含 "docs" 的目录
z docs
```

用久了之后，目录切换速度提升非常明显。支持所有主流 shell（bash、zsh、fish、PowerShell）。

### [fzf](https://github.com/junegunn/fzf)

命令行模糊查找工具。能对任何列表进行交互式模糊搜索 — 文件、Git 历史、进程、环境变量，什么都能搜。

```bash
# 模糊搜索文件并用 vim 打开
vim $(fzf)

# 模糊搜索 Git 历史
git log --oneline | fzf

# 模糊搜索并切换目录
cd $(find . -type d | fzf)
```

配合快捷键绑定，可以实现 Ctrl+T 搜索文件、Ctrl+R 搜索历史命令等高效操作。

### [starship](https://starship.rs/)

跨 shell 的命令行提示符（prompt）。显示 Git 分支、Node/Python/Rust 版本、执行时间等上下文信息，比默认提示符信息量大得多。配置简单，一个 TOML 文件搞定。

### [tmux](https://github.com/tmux/tmux)

终端复用器。一个终端窗口里开多个面板，断开 SSH 后会话不丢失。远程开发必备工具。

```bash
# 新建会话
tmux new -s work

# 断开（不关闭会话）
Ctrl+B, D

# 重新连接
tmux attach -t work
```

## 编辑器与笔记

### [Vim](https://www.vim.org/)

终端文本编辑器。学习曲线陡峭，但一旦掌握，编辑速度远超普通编辑器。不只是一个编辑器 — Vim 的键位设计是一种操作语言，很多其他工具都支持 Vim 模式（VS Code、IDEA、甚至浏览器）。

入门建议：先跑一遍 `vimtutor`（Vim 自带教程，30 分钟），然后在 VS Code 里装 Vim 插件，逐步适应。

### [Obsidian](https://obsidian.md/)

基于 Markdown 的本地知识管理工具。所有笔记都是本地 Markdown 文件，不绑定任何平台。支持双向链接、图谱视图、插件生态。

适合用来建个人知识库 — 课程笔记、技术文档、读书笔记。配合 Git 做版本控制，比 Notion 更可控。

### [Neovim](https://neovim.io/)

Vim 的现代化分支。兼容 Vim 的键位和插件，但有更好的默认配置、Lua 脚本支持和异步插件加载。如果你要从零开始学 Vim，建议直接学 Neovim。

## 桌面效率

### [Everything](https://www.voidtools.com/zh-cn/)

Windows 文件搜索神器。基于 NTFS 的 USN Journal，搜索速度是 Windows 自带搜索的几百倍。输入关键词，结果实时出现。

装上之后，找文件再也不用一层层翻目录了。建议设一个全局快捷键（比如 Ctrl+Space）随时呼出。

### [Flow Launcher](https://www.flowlauncher.com/)

Windows 效率启动器，类似 macOS 的 Spotlight / Alfred。快速启动应用、搜索文件、做计算、查天气，通过插件还能搜索书签、剪贴板历史、GitHub 等。

```bash
# 用 Scoop 安装
scoop install flow-launcher
```

默认快捷键 Alt+Space，随时呼出，用完即走。

### [PowerToys](https://learn.microsoft.com/zh-cn/windows/powertoys/)

微软官方的 Windows 增强工具集。包含窗口管理（FancyZones）、颜色选择器、批量重命名、键盘管理器、屏幕标尺等十几个实用工具。

开发者最常用的功能：
- **FancyZones**：自定义窗口布局，多显示器利器
- **PowerToys Run**：和 Flow Launcher 类似的快速启动
- **Keyboard Manager**：重映射按键

## 在 AI 时代的价值

AI 工具（如 Claude Code、Copilot）正在改变编程方式，但这些效率工具的价值不降反升：

**终端工具是 AI 编程的基础设施。** AI 辅助编程工具（如 Claude Code）运行在终端中。一个配置良好的终端环境（lsd、zoxide、fzf、tmux）能让你更快地导航、查找和操作文件，提升与 AI 协作的效率。

**本地知识管理对抗信息碎片化。** AI 时代信息爆炸，Obsidian 帮你把零散的知识点组织成可检索、可关联的知识网络。AI 擅长生成内容，但知识的积累和内化仍然需要你自己的笔记系统。

**效率工具释放认知资源。** Everything、Flow Launcher 这些工具帮你减少在"找东西"上的时间消耗。省下来的时间和注意力，可以用在更有价值的事情上 — 比如理解 AI 生成的代码，或者思考系统设计。
